# رسومات تحليل نظام Plantify

بناءً على ملف متطلبات النظام الذي قمت بإرفاقه، قمت بإعداد الرسومات الخمسة (بصيغة مخططات تفاعلية) مطابقة للصور والملفات المطلوبة:

---

## 1. مخطط السياق (Context Diagram / Level -1) 
يُظهر النظام كوحدة واحدة وكيفية تفاعل الكيانات الخارجية معه.

```mermaid
flowchart TD
    User([المستخدم - صاحب النبات])
    Admin([مدير النظام])
    Supplier([المورد])

    System((نظام <br> Plantify ))

    User -- "بيانات الحساب، تفاصيل النباتات،<br>طلبات تشخيص، طلبات شراء\n" --> System
    System -- "إشعارات التذكير، نتائج التشخيص،<br>تأكيدات الشراء" --> User

    Admin -- "أوامر إدارة الحسابات والبيانات" --> System
    System -- "تقارير وإحصائيات المنصة" --> Admin

    Supplier -- "تفاصيل المنتجات وتحديث المخزون" --> System
    System -- "طلبات الشراء الخاصة بالمورد" --> Supplier
```

---

## 2. مخطط تدفق البيانات للمستوى صفر (Level 0 DFD)
يُفكك النظام إلى عملياته الرئيسية (الوحدات الوظيفية) مع توضيح قواعد البيانات.

```mermaid
flowchart TD
    User([المستخدم])
    Admin([مدير النظام])

    P1((1. التسجيل <br>والحسابات))
    P2((2. إدارة <br>حديقتي))
    P3((3. نظام <br>التذكير))
    P4((4. التشخيص <br>الذكي))
    P5((5. المتجر <br>الإلكتروني))
    P6((6. المجتمع))
    P7((7. لوحة<br>التحكم))

    D1[(D1: المستخدمين)]
    D2[(D2: النباتات)]
    D3[(D3: المنتجات والطلبات)]
    D4[(D4: المنشورات)]

    User -->|التسجيل وتسجيل الدخول| P1
    P1 -->|تحديث البيانات| D1

    User -->|إضافة/تعديل/حذف| P2
    P2 -->|بيانات النبات| D2

    D2 -->|موعد الري| P3
    P3 -->|إشعارات التذكير| User

    User -->|صورة وبيانات| P4
    P4 -->|نتيجة التشخيص| User

    User -->|طلبات شراء| P5
    P5 -->|تفاصيل| D3
    P5 -->|تأكيد الطلب| User

    User -->|إنشاء منشور/تفاعل| P6
    P6 -->|سجل التفاعل| D4

    Admin -->|أوامر إدارة| P7
    P7 -->|إدارة| D1
    P7 -->|إدارة| D2
    P7 -->|إدارة| D3
```

---

## 3. المخطط الفرعي (Child Diagrams / Level 1 DFD) - وحدة "حديقتي"
تفصيل دقيق لوحدة إدارة النباتات (حديقتي).

```mermaid
flowchart TD
    User([المستخدم])
    D2[(D2: قاعدة بيانات النباتات)]
    P3((3. نظام التذكير))

    P2_1((2.1 إضافة نبات))
    P2_2((2.2 تسجيل تاريخ الري))
    P2_3((2.3 رفع صور دورية))
    P2_4((2.4 حذف نبات))

    User -->|بيانات النبات الجديد| P2_1
    P2_1 -->|سجل نبات جديد| D2

    User -->|تأكيد سقي النبات| P2_2
    P2_2 -->|تحديث تاريخ آخر ري| D2
    P2_2 -->|إعادة جدولة الري| P3

    User -->|صورة نمو دورية| P2_3
    P2_3 -->|حفظ الصورة| D2

    User -->|طلب حذف نبات| P2_4
    P2_4 -->|حذف السجل| D2
```

---

## 4. مخطط حالة الاستخدام (Use Case Diagram)
يوضح التفاعلات بين الفئات المستهدفة والنظام ككل.

```mermaid
flowchart LR
    User([مستخدم - صاحب نبات])
    Admin([المدير - Admin])

    subgraph Plantify
        direction TB
        UC1(إنشاء حساب وتسجيل الدخول)
        UC2(إدارة النباتات - حديقتي)
        UC3(استقبال تذكيرات الري التلقائية)
        UC4(تشخيص أمراض النباتات بالذكاء الاصطناعي)
        UC5(شراء المنتجات من المتجر)
        UC6(نشر ومتابعة في مجتمع محبي النباتات)
        
        UC7(إدارة بيانات المستخدمين)
        UC8(إدارة النباتات والمنتجات)
        UC9(إدارة طلبات المتجر)
        UC10(مراجعة وإصدار الإحصائيات)
    end

    User --> UC1
    User --> UC2
    User --> UC3
    User --> UC4
    User --> UC5
    User --> UC6

    Admin --> UC7
    Admin --> UC8
    Admin --> UC9
    Admin --> UC10
```

## 5. مخطط التسلسل (Sequence Diagram) - عملية التشخيص الذكي
تتبع ترتيب العمليات الزمني لوحدة التشخيص الذكي (حيث يرفع المستخدم صورة للنبات المريض).

```mermaid
sequenceDiagram
    actor U as المستخدم
    participant UI as واجهة النظام (UI)
    participant B as الخادم (Backend)
    participant AI as مساعد الذكاء الاصطناعي
    participant DB as قاعدة البيانات

    U->>UI: يرفع صورة النبات للتشخيص
    activate UI
    UI->>UI: التحقق من ملائمة الصورة
    UI->>B: يرسل طلب تشخيص (API)
    activate B
    B->>AI: يرسل الصورة للتحليل والإفادة
    activate AI
    AI-->>AI: معالجة الصورة واكتشاف الأمراض/الآفات
    AI-->>B: يُرجع (المرض + مقترح العلاج + نسبة الدقة)
    deactivate AI
    B->>DB: يحفظ نتيجة التشخيص بسجل النبات لتوثيق الحالة
    activate DB
    DB-->>B: تأكيد الحفظ
    deactivate DB
    B-->>UI: يرسل استجابة بالنتيجة
    deactivate B
    UI-->>UI: تنسيق النتيجة للمشاهدة
    UI-->>U: يعرض للمستخدم نتيجة التشخيص ومقترحات العلاج
    deactivate UI
```
